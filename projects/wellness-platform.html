What This Is

A full stack gamified wellness application built in 24 hours for the Star Union Dai ichi National Tech Hackathon 2025. The team ranked 9th of 100 competing teams.

The platform maps personal health across five dimensions including mind, body, social, financial, and purpose as an interconnected node graph. Completing activity in one dimension propagates a ripple effect to connected dimensions. The gamification layer applies behavioral psychology directly to retention mechanics. It targets three specific failure modes identified in the brief including low feature exploration depth, drop off after the first week, and the absence of social accountability loops.

The Wellness Graph

The core technical metaphor is a 6 node interconnection graph where mind has two sub nodes. Each dimension holds a gem with a power level. When you complete an activity like a workout or a social challenge, the gem in that dimension powers up. The ripple effect engine then propagates a 30 percent energy gain to all directly connected dimensions. This is computed in linear time per update where n is the number of connected edges.

This is not cosmetic. The interconnection rules encode a product thesis that wellness habits compound. If you exercise, your sleep improves. If your social connections are strong, your sense of purpose is clearer. The graph structure makes those dependencies explicit and interactive.

The decay mechanic makes the reverse true as well. When gems are neglected they lose power. If the body gem decays to dim, the mind gem automatically shifts to draining and loses an additional 10 power points on top of base decay. The graph is always moving in one direction or the other.

Architecture Under Hackathon Constraints

Three services are coordinated via Vite dev proxy. These include a React frontend on port 5173, an Express REST API on port 5001, and a Flask ML microservice on port 5002. The frontend operates fully without the backend via client side simulation in the API module. Only the analytics forecast and quest recommendations require the ML service.

This was a deliberate architectural decision. Services go down in a 24 hour hackathon. The frontend needed to be demonstrable regardless of whether the backend was running. Local storage based persistence meant a page refresh did not wipe state. The client side TF.js fallback meant the analytics dashboard was never blank.

State Management Without External Libraries

The entire global state including gem power levels, quest completion status, XP, active notifications, and unlocked achievements is managed by a single 5 action useReducer in the Wellness Context. There are five action types.

  • COMPLETE QUEST marks quest complete, increases gem power by 20, and awards XP.
  • SIMULATE DECAY applies daily power decay with cross gem cascade rules.
  • RESET GAUNTLET resets all gems to initial state and preserves XP.
  • ADD NOTIFICATION pushes to notification feed with a maximum of 5 retained.
  • UNLOCK ACHIEVEMENT unlocks milestone by ID.

This single reducer drives 10 route level page components and 5 reusable components via Context API with zero prop drilling. Redux or Zustand would have worked but added setup time and external dependencies in a 24 hour build.

A native CustomEvent bus decouples the 4 independent page components from the central API service. Components emit events and the API service listens and updates state. There is no shared mutable state, no prop chains, and no circular dependencies.

The Dual ML Pipeline

Server side, a Flask microservice fits a Facebook Prophet model to 90 days of synthetic wellness history shaped as a sine wave with noise. It forecasts 7 days ahead and returns date and predicted balance pairs.

Client side, a TF.js linear regression runs on the same historical data when the backend is unavailable. The model fits a single variable regression on the time index and extrapolates 7 days forward. It is less sophisticated than Prophet because it cannot model weekly seasonality or trend changepoints. However, it always produces an output.

The analytics dashboard exposes both predictions simultaneously via a toggle button. This lets users compare them. In a real production deployment the synthetic history would be replaced with actual user activity data from the database.

Behavioral Psychology, Implemented

Habit Formation

Daily quests are small, achievable, and provide immediate visual feedback via a gem glow animation on completion. The feedback loop is tight.

Loss Aversion

Gems lose power when neglected. Users return not just to gain progress but to prevent losing what they have. Loss aversion is roughly twice as powerful a motivator as equivalent gain.

Endowed Progress

The Mind Stone starts at 15 percent power. Users feel already invested from the first login. Starting at 0 creates a blank slate that is easy to abandon. Starting at 15 percent creates ownership.

Variable Reward Schedules

Three quest difficulty tiers with different XP multipliers prevent habituation. Fixed rewards become expected and lose motivational force. Variable rewards maintain engagement indefinitely.

Social Accountability

The Soul Gem requires community activity to power up. A friend leaderboard shows peer progress. Social accountability is the strongest long term retention mechanism. It is harder to implement, but the product explicitly surfaces it.
