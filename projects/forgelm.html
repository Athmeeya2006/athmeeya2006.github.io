ForgeLM

VIEW CODE →

What It Does

Trains a GPT-style language model in PyTorch and runs its C++ CPU inference engine: matmuls, attention, KV-cache, int4 quantizer, sampler, and HTTP server. Accelerates decoding from 20 to 6,359 tokens per second, a 312x gain, with a KV-cache and AVX2/FMA SIMD kernels, output staying bit-identical to the PyTorch reference. Adds int4 quantization for 7.1x smaller weights and a paged KV-cache serving 16 concurrent requests at 8,362 tokens per second.

The model is a byte-level GPT with 858,880 parameters, roughly 0.86M, using 4 layers, 4 attention heads, an embedding width of 128, and a block size of 256. PyTorch trains it. A separate C++ engine serves it on a CPU, which is where the throughput work happens.

How It Works

From Naive Loop to SIMD Kernels

Decode throughput is measured on a single core with deterministic greedy decoding so every variant produces the same tokens. The naive loop runs at 20.38 tokens per second. Adding a KV-cache, which stops the model from recomputing attention over the whole prefix every step, lifts it to 2,620 tokens per second, a 128.6x gain that stays bit-identical to the naive output. Adding AVX2/FMA float kernels lifts it further to 6,359 tokens per second, 312x over the naive loop and still bit-identical.

int4 Quantization

An int4 quantizer with a group size of 64 shrinks the weights 7.11x, from 3200 KiB to 450 KiB. Perplexity rises only 1.37%, so most of the memory comes off at a small accuracy cost.

Speculative Decoding

A small draft model of 133K parameters proposes tokens with K=2 that the full model verifies. The output stays bit-identical to greedy decoding, and the draft tokens are accepted 50.6% of the time.

Paged KV-Cache and Batching

A paged KV-cache with continuous batching serves 16 concurrent requests at 8,362 tokens per second aggregate, a 1.21x throughput gain under load over serving them without paging.

Results

Decoding goes from 20.38 to 6,359 tokens per second on one core, a 312x gain, with the optimized output staying bit-identical to the PyTorch reference at every stage.

int4 quantization cuts weight size 7.11x for a 1.37% perplexity increase. Speculative decoding accepts 50.6% of draft tokens with identical output, and the paged KV-cache serves 16 concurrent requests at 8,362 tokens per second.
