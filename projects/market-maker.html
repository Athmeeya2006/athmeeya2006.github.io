No Regret Market Making Engine

VIEW CODE →

What This Is

A full stack research sandbox for studying no regret learning in market making. It combines a C++17 limit order book with Python bindings via pybind11. It includes adversarial and stochastic order flow generators. An evaluation toolkit benchmarks 6 algorithms across 10,000 round simulations under stationary and adversarial market conditions.

The central question asks whether a spread quoting strategy can guarantee bounded regret under parameter drift and adversarial order flow without environmental models. Classical market making assumes calibrated arrival models and stationary volatility. Regimes shift and parameters drift in practice. Adversarial traders exploit fixed strategies. The Exp3 algorithm family makes no such assumptions. They guarantee sublinear regret even under adversarial rewards. This project stress tests that guarantee across 4 market regimes with abrupt and gradual transitions.

The C++17 Order Book

The matching engine is 760 lines of C++17 with price time priority and FIFO queues at each price level. Price levels use standard maps for logarithmic insertion and lookup. Individual queues at each level use double ended queues for time priority ordering. A separate hash map order index enables constant time cancel lookups.

The engine exposes derived statistics that feed directly into the learning algorithms. It computes order flow imbalance and rolling realized volatility from logarithmic return standard deviation over a sliding window. It tracks volume weighted average price and multi level depth queries for both bid and ask sides.

The pybind11 binding layer maps C++ methods to Python with automatic standard template library container conversion. It exposes the engine as a Python module. A minimal Python stub provides a fallback for lightweight testing if the C++ extension is unavailable. This stub is slower but sufficient to run the learning algorithms without rebuilding.

Six Algorithms, One Simulation Loop

Each round of the simulation proceeds in eight steps. It applies the current regime and builds a 9 dimensional context vector from book state and inventory. It queries the market maker for a spread and submits quotes. It samples and submits noise and informed orders. It advances the mid price by a geometric Brownian motion step. It computes reward and counterfactual rewards for all spread arms. Finally it updates the market maker.

Exp3

The standard adversarial bandit uses importance weighted reward updates with an optimal learning rate. It achieves the theoretical regret bound. Per round regret decreases over time. This was verified empirically at 3,000 rounds.

Exp3 with Doubling Trick

Restarts Exp3 on doubling epochs to handle unknown horizons. This costs a constant factor penalty in the regret bound but requires no horizon parameter at initialization.

Sliding Window Exp3

Forgets rewards older than the specified window. This makes it non stationary. It tracks regime shifts by forgetting the past rather than averaging over the full history. It recovers the correct spread arm faster than standard Exp3 after regime changes. It suffers higher variance during stationary periods.

Contextual Bandit EXP4

Mixes a pool of 8 experts including fixed spread, volatility scaled, inventory aware, order flow imbalance, and momentum. It reweights experts continuously and concentrates weight on the dominant expert per regime. The context vector provides the 9 dimensional feature input that differentiates EXP4 from context free Exp3 variants.

Avellaneda and Stoikov, 2008

The model based baseline uses a closed form spread derived from the reservation price formula. It applies maximum likelihood estimation calibration for volatility and order arrival rate. It outperforms Exp3 under stationary conditions where the model is well calibrated. It degrades measurably when parameters shift between regimes and the calibration becomes stale.

Fixed Spread Baselines

Eight fixed spread instances anchor the regret computation. They provide the counterfactual reference for comparing all other algorithms.

Three Order Flow Models

NoiseTrader

Generates Poisson arrivals with power law order sizes. It produces balanced and price insensitive order flow. This represents the benign background flow that generates spread revenue.

StochasticInformedTrader

Trades when the true asset value exceeds the quoted price. Order size scales with the edge between value and quote. Arrival rate is 20 percent of the noise rate. This is adverse selection where every fill from an informed trader is a loss.

AdversarialInformedTrader

Observes the market maker spread distribution and computes a best response. It alternates between exploit phases and bait phases. Exploit phases submit large informed orders when quotes are tight. Bait phases submit small balanced orders to pull quotes tighter. This is the hardest adversary in the system and the primary stress test.

The Counterfactual Approximation

Regret computation requires knowing what reward each spread arm would have earned in every round. Exact computation would require re running the full simulation for each arm per round. This computational cost makes 10,000 round benchmarks impractical.

The simulator uses an exponential fill approximation instead. Order arrival sensitivity to spread width is modeled exponentially. This approximation is directionally correct because tighter spreads receive more fills. It remains biased relative to the true fill distribution from the order flow.

This is documented explicitly as a design limitation. The regret curves measure performance relative to the approximated counterfactual rather than an oracle. The no regret guarantee holds under this approximation. The absolute regret values should be interpreted with that caveat in mind.

Evaluation Toolkit

RegretTracker computes exact external regret versus the best fixed spread in hindsight. It uses per round counterfactuals across all spread arms. Rolling regret over a 500 round window and bootstrap confidence intervals make the convergence curves statistically interpretable.

RegretDecomposer breaks economic profit and loss into three components. It separates spread revenue from adverse selection loss and inventory loss. It tracks cumulative profit by component separately for each algorithm.

AdaptationSpeedAnalyzer measures recovery time after each regime change. It compares empirical recovery to theoretical predictions. It computes Pearson correlation between predicted and observed recovery times with statistical significance testing.

Benchmarker runs all learning algorithms and fixed spread baselines on a shared simulation factory. It reports total reward, total regret, theoretical bounds, Sharpe ratio, max drawdown, and mean absolute inventory.

Four Market Regimes

The Calm regime features low volatility and normal arrival rates. The Volatile regime features high volatility and elevated arrival rates. The Illiquid regime features low arrival rates and high adverse selection. The Informed regime features high informed fraction with adversarial order flow.

Transitions between regimes are configurable as abrupt or gradual. The default duration is 2,000 rounds per regime across a 10,000 round simulation.

This regime structure makes the benchmark meaningful. Any algorithm can perform well in stationary conditions. The interesting question is how quickly it adapts when the environment changes. The simulation tests whether its regret bound still holds during and after the transition.

GARCH Volatility and Synthetic Microstructure

The true value process is geometric Brownian motion with optional Poisson jumps. The observed mid price is a mean reverting perturbation of true value that creates microstructure noise. Volatility clustering comes from a GARCH process applied to true value returns. Order sizes follow a power law distribution with fat tails.

Real market data adds engineering overhead and legal complexity without improving the core property being tested. Synthetic data with controlled statistical properties makes the experiment reproducible. It makes regime transitions configurable and adversarial scenarios exact.

Tests and CI

Eight test modules cover algorithmic convergence and matching engine mechanics. They verify regime adaptation, expert pool weight concentration, and adversarial trader logic. They validate the reservation price formula, regret decomposition component accounting, and numerical stability.

GitHub Actions builds the C++ extension from source. It validates the engine import and runs the full test suite on every push. Build and test complete in under two minutes.
